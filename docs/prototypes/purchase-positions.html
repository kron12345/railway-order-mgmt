BESTELLPOSITIONEN

3 Konzepte fuer die Darstellung von 100+ Bestellpositionen pro Auftragsposition (z.B. taeglicher Zug ueber 3 Monate)

1 — Kalender-Heatmap

Empfohlen
Idee: Jeder Tag = ein farbiges Kaestchen. Farbe = Bestellstatus. Hover zeigt Details. Wie eine GitHub-Contribution-Map, aber fuer Bestellungen. Sofort erkennbar: wo gibt es Probleme?
Pro: 90 Tage auf einen Blick, Muster sofort erkennbar (z.B. "Woche 3 alle abgelehnt"), Tooltip fuer Details.
Contra: Einzelne Bestellung schwer zu editieren, eher Monitoring als CRUD.

2 — Zusammengefasste Zeitbloecke

Kompakt
Idee: Aufeinanderfolgende Tage mit gleichem Status werden zu Bloecken zusammengefasst. Statt 90 Zeilen sieht man z.B. 8 Bloecke: "01.12–22.12: 22× Bestaetigt", "23.12–25.12: 3× Abgelehnt", etc. Klick auf Block klappt die einzelnen Tage auf.
Pro: Dramatische Reduktion (90 → ~8 Zeilen), CRUD-faehig beim Aufklappen.
Contra: Verliert Tagesmuster, Wochenenden nicht direkt sichtbar.

3 — Timeline-Streifen + Filter-Tabelle

Hybrid
Idee: Oben ein farbiger Zeitstreifen als Ueberblick (proportionale Segmente nach Status). Darunter eine scrollbare Tabelle mit Status-Filtern. Klick auf ein Segment im Streifen filtert die Tabelle.
Pro: Bester Kompromiss aus Ueberblick und CRUD, Filter reduzieren sichtbare Zeilen.
Contra: Braucht mehr Platz als die anderen, zwei Interaktionszonen.
