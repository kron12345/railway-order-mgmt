ORDER VIEW KONZEPTE

3 Design-Ansaetze fuer Status-Aggregation und Inline-Positionen — Railway Order Management

A — Expandable Rows + Status-Balken

Empfohlen
Idee: Jede Auftragszeile hat einen farbigen Statusbalken der die Positionsstatus aggregiert zeigt. Klick auf den Chevron klappt die Positionen als Sub-Rows inline auf. Bloomberg-Terminal-Stil: kompakt, datengetrieben, schneller Ueberblick.
Pro: Schnellster Ueberblick, keine Navigation noetig, skaliert gut bei vielen Auftraegen.
Contra: Begrenzte Detailtiefe in den Sub-Rows.

B — Master-Detail Split View

Detailreich
Idee: Links die Auftragsliste (kompakt), rechts ein Detailpanel mit allen Positionskarten. Klick auf einen Auftrag laed die Details im rechten Panel, ohne Navigation.
Pro: Maximale Detailtiefe, Ressourcen-Chips sichtbar, Positionen als Karten gut strukturiert.
Contra: Braucht Bildschirmbreite, weniger Auftraege gleichzeitig sichtbar.

C — Accordion + Status-Heatmap + Kacheln

Visuell
Idee: Jeder Auftrag zeigt eine Mini-Heatmap der Positionsstatus als farbige Zellen. Aufklappen zeigt Positionen als 2-spaltige Kacheln mit Ressourcen-Icons. Sehr visuell und auf einen Blick erfassbar.
Pro: Am intuitivsten, Heatmap zeigt sofort Problemstellen, Kacheln geben mehr Platz fuer Details.
Contra: Braucht mehr vertikalen Platz wenn ausgeklappt.
